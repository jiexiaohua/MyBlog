# 个人博客第一版的构建取舍

个人博客最重要的不是功能堆满，而是能长期写下去。

这一版选择了几个朴素的方案：

1. 文章用 Markdown 文件保存，方便备份、迁移和 Git 追踪。
2. 页面用 Next.js App Router 构建，公共访问走服务端渲染。
3. 发文接口只验证管理员会话，不做复杂的多用户系统。
4. 部署在自己的服务器上，由 Nginx 代理到 Node 进程。

这个组合的好处是维护成本低，后续要扩展搜索、RSS、评论或图片上传时，也可以在现有结构上继续加。
